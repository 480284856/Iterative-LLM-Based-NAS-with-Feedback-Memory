
1. Easy Understanding (Informal Definition)

    Imagine you ask an AI to write a story. Instead of just giving you one answer, the AI writes 10 different versions (the "N-best" hypotheses).Some versions might be mostly bad but have one perfect sentence. Other versions might be great but use one wrong word.
    
    Instead of just picking the "least bad" version, this method acts like an editor. It looks at every word in all 10 stories and decides: "This word is high-quality, keep it" (Positive Constraint) or "This word is a mistake, avoid it" (Negative Constraint). It then takes all those "good fragments" and stitches them together to create a new, final version that is better than any of the original 10.

2. Formal Definition

    The goal is to generate a final sequence $Y$ that maximizes quality by ensembling a set of hypotheses $\mathcal{H} = \{h_1, h_2, \dots, h_n\}$ generated from an input $X$.

    The Quality Estimation (QE) Function
    For every token $y_{i,j}$ in hypothesis $h_i \in \mathcal{H}$, a binary quality estimator $f_{QE}$ predicts a label $L \in \{1, 0\}$, where $1$ represents a token to include and $0$ represents a token to avoid:

    $$f_{QE}(X, h_i, y_{i,j}) \rightarrow L \in \{1, 0\}$$
    Constraint Composition
    The model determines the final set of constraints through majority voting across all hypotheses:

    Positive Constraints ($C_{pos}$): Tokens frequently predicted as high-quality across the N-best list.

    Negative Constraints ($C_{neg}$): Tokens frequently predicted as low-quality/incorrect.

    The Decoding Objective
    The final output $Y^*$ is generated by a language model $P(Y|X)$ while satisfying the set of constraints $\mathcal{C} = \{C_{pos}, C_{neg}\}$ using the NeuroLogic-A* algorithm:

    $$Y^* = \text{argmax}_Y \left( \log P(Y|X) + \lambda \cdot \text{Satisfaction}(Y, \mathcal{C}) \right)$$

    (Note: $\lambda$ represents the weight given to satisfying the constraints vs. the standard model probability).
